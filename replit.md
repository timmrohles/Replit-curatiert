## Overview
coratiert.de is a curated book recommendation platform for personalized book discovery, combining a robust backend with a dynamic frontend. It aims for high performance, user-friendliness, intelligent recommendations, and data privacy, with a focus on content curation. Key capabilities include comprehensive book data management, dynamic page generation, administrative tools, and advanced content performance tracking. The project seeks to offer a unique, curated experience in the book recommendation market.

## System Architecture
The platform is built with a modern web stack, featuring a React frontend and an Express.js backend.

**Frontend (client/)**:
-   **Framework**: React + Vite (TypeScript)
-   **Styling**: Tailwind CSS with a custom theme and self-hosted fonts. Uses CSS custom properties for all design tokens (spacing, radius, typography, colors, shadows) and provides dark mode overrides. Design system components are built on shadcn UI primitives. Microcopy uses i18n keys and adheres to a `COPY_TONE` style guide.
-   **Dynamic Content**: `UniversalSectionRenderer` and `sectionRegistry` enable dynamic rendering of content sections. `CMSHomepage` and `DynamicPage` handle CMS-driven content.

**Backend (server/)**:
-   **Framework**: Express.js
-   **Database**: Neon PostgreSQL.
-   **API**: Provides public data, admin CRUD operations, and content tracking.
-   **Authentication**: Provider-agnostic OIDC system with `openid-client` and `passport`, supporting multiple OIDC providers and role-based access control (`user`, `admin`, `super_admin`), including impersonation.

**Core Features & Design Patterns**:
-   **Dynamic Content Management**: Pages and sections are rendered based on database configurations, supporting both explicit and query-based book resolution. Category pages (`page_type='category'`) automatically filter all sections by the page's `category_id` — book_carousel sections get category-filtered queries in the resolve endpoint, and client-side sections (book_grid_filtered, user_curations) receive categoryId as a prop from DynamicPage.
-   **Book Enrichment & Badge System**: A unified badge system (awards, indie publisher, reviews) is used across all book displays, ensuring consistent presentation and detailed modals on click. Badges are generated by the `BookEnrichmentBadges` component, with data sourced from `enrichBooksWithAwards()` and `book_onix_tags`.
-   **Book Score Calculation**: An automated system calculates `award_score`, `media_score`, `curation_score`, `base_score`, `total_score`, `is_indie`, and `is_hidden_gem` for all books. It runs via cron, event-based triggers, and an admin endpoint.
-   **Book Sorting Modes**: Backend supports various sorting modes like `relevance`, `newest`, `most-awarded`, `popular`, `manual`, and `hidden-gems`.
-   **Content Scheduling & Tracking**: Features for scheduling content, setting view/click limits, and bot filtering for analytics.
-   **Curator Management**: Integrated curator selection and profile management.
-   **Type Safety**: Extensive use of TypeScript with Zod for API validation.
-   **Admin Panel**: Comprehensive Content Manager for administrative tasks.
-   **Navigation Management**: Admin UI for managing navigation items.
-   **Author Verification Workflow**: User-initiated author access requests with admin approval and ONIX database matching.
-   **Internationalization (i18n)**: Directory-based locale prefixes (`/de-de/`, `/en-gb/`) with `de-de` as default. Translation files cover numerous namespaces.

-   **User Bookstore & Curations System**: Users can create thematic book collections (Kurationen) and public bookstore profiles.
-   **Events System**: Users create and manage events.
-   **Category Cards System**: Admin-managed category cards for homepage display.
-   **Affiliate Creator Program**: Users register for an affiliate program with identity, tax, and payout details, including creator analytics.
-   **Content/Podcast Book Extraction System**: AI (OpenRouter/Mistral) extracts book mentions and sentiment from RSS feeds, with a matching system to link to database entries.
-   **Affiliate Tracking System**: Platform-internal creator links with click tracking and session-based attribution.
-   **Revenue-Share Attribution System**: Dual-source attribution for curator commissions (REFERRAL and CURATION) with priority rules.

**Security**:
-   **Security Headers**: Helmet middleware for CSP, X-Frame-Options, HSTS.
-   **Rate Limiting**: On login and API endpoints.
-   **Upload Protection**: Authenticated uploads.
-   **SQL Injection Prevention**: Whitelisting for slug generation.
-   **XSS Prevention**: DOMPurify for content sanitization.
-   **Admin Route Protection**: `requireAdminGuard` for administrative endpoints.

**SEO**:
-   **robots.txt**: Dynamically served, allowing search engines and blocking admin paths/AI crawlers.
-   **Sitemap**: Dynamic XML sitemap for static, CMS, and profile pages.
